3. **Identify the Pattern** — State which algorithmic pattern applies (e.g., sliding window, two pointers, BFS/DFS, dynamic programming, binary search, greedy, hash map).
4. **Explain Your Approach** — Walk through the algorithm in plain English before coding. Mention key data structures.
